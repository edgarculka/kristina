# Voice and typography

This site should feel calm, friendly, and professional. The copy should make it easy for visitors to understand the work, choose a service, and ask for help without feeling pushed.

## Tone of voice

- Write with warmth and clarity.
- Prefer simple, direct sentences over abstract claims.
- Sound supportive, not sales-led.
- Use "you" when it helps the visitor understand their next step.
- Keep calls to action specific and low-pressure.

## Typography and casing

- Do not use all-caps styling for visible interface text.
- Use sentence case for headings, buttons, navigation, labels, and supporting copy.
- Keep proper names and established acronyms in their standard form, such as Kristina Culka and NLP.
- Avoid decorative casing as a hierarchy tool; use font size, weight, spacing, and layout instead.
- Use short headings that describe the content plainly.
- Use the shared typography tokens in `assets/css/main.css` for font families, font sizes, line heights, tracking, and readable text measures.
- Keep body copy around 60-70 characters per line where layout allows.
- Use normal kerning and ligatures. Do not add negative letter spacing.
- Avoid viewport-width font sizing; adjust type through semantic tokens and breakpoints.
- Use tighter line heights for display text and roomier line heights for paragraphs, form help text, and list copy.

## Examples

- Use "How I work" instead of "How I Work".
- Use "Three clear service paths" instead of "Three Clear Service Paths".
- Use "Book a consultation" instead of "Book a Consultation".
- Use "Available in London and online." instead of terse label-style fragments where a full sentence feels more human.
